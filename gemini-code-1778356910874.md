# 專案規格書：Floorplan 光柵圖轉完美向量骨架 (Raster to Vector Skeleton)

## 1. 任務目標

請實作一個 Python 腳本，將帶有厚度的 Floorplan 光柵圖（Raster Image）精準轉換為「1 像素粗細的中心線向量圖（Vector Skeleton）」，並匯出成結構化的 JSON 資料。

**最高指導原則：完全不考慮運算效能與時間，只追求「完美無瑕的數學精度」與「轉角處/交界處的完美閉合」，絕對不能有毛刺或斷線。**

## 2. 系統環境與依賴套件

- **核心套件:** `opencv-python` (cv2), `scikit-image`, `numpy`, `networkx`

## 3. 輸入與輸出定義

- **輸入:** 一張 Floorplan 圖片 (例如 `.png`)。
  - 背景色：白色。
  - 有意義的線條顏色：黑色 (Wall)、藍色 (Window)、黃色 (Door)。線條具有不等寬的粗細。
- **輸出:** 一個 `output.json` 檔案。
  - 格式範例：
    ```json
    {
      "lines": [
        { "type": "wall", "x1": 10.5, "y1": 20.0, "x2": 100.5, "y2": 20.0 },
        { "type": "window", "x1": 100.5, "y1": 20.0, "x2": 150.0, "y2": 20.0 }
      ]
    }
    ```

## 4. 核心演算法步驟要求 (請嚴格遵循)

### 步驟一：精準的顏色分離 (Color Segmentation)

請使用 HSV 或 RGB 色彩空間，將圖片中的黑色、藍色、黃色精準提取出來，生成三個獨立的二值化 Mask (背景為黑，線條為白)。請注意處理邊緣抗鋸齒造成的過渡色像素。

### 步驟二：學術級骨架化 (Skeletonization)

**警告：絕對禁止使用一般的 Canny 邊緣檢測。**
請使用 `scikit-image` 的 `morphology.skeletonize` (中軸變換相關演算法)，對三個 Mask 分別進行骨架化，確保提取出數學上絕對置中的 1 像素中心線，並將轉角處的毛刺 (Artifacts) 降到最低。

### 步驟三：直線提取與次像素擬合 (Line Extraction)

對骨架化後的像素點進行尋點與直線擬合。可以使用 `cv2.HoughLinesP` 或輪廓多邊形擬合 (`cv2.approxPolyDP`)。如有必要，請加入次像素 (Sub-pixel) 精度的優化，確保斜線的座標精確度。

### 步驟四：圖論級別的節點完美融合 (Topological Node Snapping) 🌟 關鍵

這是本專案的靈魂。從影像處理提取出的線段一定會有微小的斷裂或未完美交接。
請使用 `NetworkX` 建構圖論模型 (Graph)：

1. 將所有線段的起點與終點視為 Nodes，線段視為 Edges。
2. 設定一個容差半徑 (例如 5~10 pixels)。
3. 找出距離在容差範圍內的所有 Nodes，強制將它們的座標進行平均/合併 (Merge)，確保 T 字路口、L 型牆角、以及不同顏色線條 (例如黑牆接藍窗) 的端點**共用同一個絕對數學座標**。
4. 根據合併後的新 Node 座標，重新更新所有線段 (Edges) 的 `x1, y1, x2, y2`。

## 5. 驗證與繪圖 (除錯用)

在產出 JSON 的同時，請額外寫一個 `draw_debug_image()` 函數，利用提取出的 JSON 向量資料，使用 OpenCV 或 matplotlib 畫一張預覽圖並存檔，方便我肉眼比對結果是否完美閉合。
